# whoisarjen.com — Portfolio & Blog

## Project Overview

Personal portfolio and blog built with Next.js. Displays GitHub projects automatically via the GitHub API, with live screenshots generated by Microlink.

## Architecture

- **GitHub projects** are fetched at runtime from the GitHub API (`lib/github.ts`)
- Projects with a `homepage` URL get automatic screenshots via Microlink API (`getScreenshotUrl`)
- Display order and overrides are configured in `data/projectsData.ts`
- Extra (non-GitHub) projects use static images in `public/static/images/projects/`

## Before Pushing

When adding or updating projects, always do the following before pushing:

1. **Update `data/projectsData.ts`** — add the project to `displayOrder` array and add any `overrides` if needed
2. **Update `github-profile/README.md`** — add the project to the GitHub profile Projects table
3. **Regenerate project screenshots locally** — for any project with a live site, capture a fresh 1280x720 screenshot and save it to `public/static/images/projects/project-<name>.png`. This ensures the portfolio has up-to-date images even if Microlink cache is stale.
4. **Verify the homepage URL** is set on the GitHub repo (`gh repo edit <repo> --homepage "<url>"`) so the automatic screenshot works at runtime too.

## Key Files

- `data/projectsData.ts` — project display order, overrides, and extra (non-GitHub) projects
- `lib/github.ts` — GitHub API fetch + Microlink screenshot URL generation
- `github-profile/README.md` — GitHub profile README (the one shown on github.com/whoisarjen)
- `public/static/images/projects/` — static project screenshot images
